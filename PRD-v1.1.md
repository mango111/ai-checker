# PRD - AI Checker 盈利功能 v1.1

## 1. 背景

AI Checker v1.0 已上线，需要增加盈利功能实现商业化。

## 2. 目标

- 建立 Freemium 模式
- 实现付费转化
- 追踪用户行为

## 3. 功能需求

### 3.1 免费限制（P0）
- 未登录用户：每天 3 次检测（localStorage 记录）
- 超出限制显示付费引导

### 3.2 用户系统（P0）
- 邮箱注册/登录
- Google 一键登录（可选，后续）
- 用户检测次数记录

### 3.3 付费套餐（P0）
| 套餐 | 价格 | 次数 | 功能 |
|------|------|------|------|
| 免费 | ¥0 | 3次/天 | 基础检测 |
| 基础 | ¥29/月 | 50次/月 | + 历史记录 |
| 专业 | ¥99/月 | 无限 | + PDF导出 + 监控 |

### 3.4 支付集成（P0）
- 支付宝（国内）
- Stripe（海外，可选）

### 3.5 历史记录（P1）
- 保存检测历史
- 查看历史报告
- 对比不同时间的评分

### 3.6 PDF 导出（P1）
- 导出检测报告为 PDF
- 包含评分、建议、AI 视角

### 3.7 数据统计（P0）
- Google Analytics 集成
- 追踪：访问量、检测次数、付费转化

## 4. 技术方案

### 4.1 数据库
- Supabase（免费额度够用，自带 Auth）

### 4.2 支付
- 支付宝当面付 / 网页支付
- 或 LemonSqueezy（海外，简单）

### 4.3 部署
- 继续用 Vercel

## 5. MVP 范围（本次）

**本次实现：**
- [x] 免费次数限制（3次/天）
- [x] 付费引导弹窗
- [x] Google Analytics
- [ ] 用户系统（下版本）
- [ ] 支付集成（下版本）

**原因：** 先验证有没有人用，再加复杂功能

## 6. 验收标准

- [ ] 免费用户每天限 3 次
- [ ] 超出显示付费引导
- [ ] GA 能看到访问数据
- [ ] 不影响现有功能

| 阶段 | 时间 |
|------|------|
| 开发 | 30 分钟 |
