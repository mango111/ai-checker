## 1. 产品概述

### 1.1 背景
随着 ChatGPT、Claude、Perplexity 等 AI 搜索/对话工具的普及，网站的"AI 可见性"成为新的 SEO 方向。网站主需要知道 AI 模型如何"看"自己的网站，以便优化内容。

### 1.2 目标
提供一个工具，让用户输入网址，展示 AI 模型看到的网站内容，并给出优化建议。

### 1.3 目标用户
- 网站站长
- SEO 从业者
- 内容创作者
- 独立开发者

## 2. 用户故事

### US-001 检测网站 AI 可见性
**作为** 网站站长
**我想要** 输入我的网址，看到 AI 模型如何理解我的网站
**以便于** 优化网站内容，提升 AI 搜索排名

**验收标准：**
- [ ] 输入网址后，显示 AI 看到的内容
- [ ] 显示页面标题、描述、主要内容
- [ ] 显示结构化数据（如有）
- [ ] 响应时间 < 10 秒

### US-002 获取优化建议
**作为** SEO 从业者
**我想要** 获得针对性的 AI 优化建议
**以便于** 改进网站的 AI 可见性

**验收标准：**
- [ ] 显示优化建议列表
- [ ] 建议分优先级（高/中/低）
- [ ] 每条建议有具体说明

### US-003 对比多个 AI 模型视角
**作为** 内容创作者
**我想要** 看到不同 AI 模型（ChatGPT/Claude）的视角差异
**以便于** 针对性优化

**验收标准：**
- [ ] 支持选择不同 AI 模型
- [ ] 并排对比展示
- [ ] 标注差异点

## 5. 技术方案建议

### 5.1 技术栈
- 前端：Next.js 14 + Tailwind CSS
- 后端：Next.js API Routes 或 FastAPI
- 部署：Vercel

### 5.2 核心逻辑
1. 接收用户输入的 URL
2. 使用 fetch/puppeteer 抓取页面内容
3. 解析 HTML，提取：
   - title、meta description
   - h1-h6 标题结构
   - 主要文本内容
   - 结构化数据（JSON-LD）
   - Open Graph 标签
4. 根据规则计算评分
5. 生成优化建议

### 5.3 评分规则（示例）

| 检查项 | 权重 | 说明 |
|--------|------|------|
| 有 title | 15 | 标题存在且长度合适 |
| 有 meta description | 15 | 描述存在且长度合适 |
| 有 h1 | 10 | 页面有且仅有一个 h1 |
| 标题层级合理 | 10 | h1→h2→h3 层级清晰 |
| 有结构化数据 | 15 | JSON-LD 或 Microdata |
| 有 Open Graph | 10 | og:title, og:description 等 |
| 内容长度 | 10 | 主要内容 > 300 字 |
| 图片有 alt | 10 | 图片有替代文本 |
| 链接有描述 | 5 | 链接文本有意义 |

## 6. MVP 范围

### 6.1 包含
- 单页面检测
- 基础评分（0-100）
- 5-10 条优化建议
- 响应式设计（PC + 移动端）

### 6.2 不包含
- 用户登录/注册
- 历史记录持久化
- 多模型对比
- 批量检测
- API 接口

### 6.3 预估工时
- 需求分析：0.5 天 ✅
- 架构设计：0.5 天
- 前端开发：1 天
- 后端开发：1 天
- 测试验收：0.5 天
